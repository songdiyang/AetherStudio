## 简介
本指南面向首次接触牧羊人编辑器的开发者，帮助你在 Windows 平台上快速搭建开发环境、完成构建并运行第一个实例。内容涵盖环境准备、构建命令、运行方式、产物路径与常用场景（直接启动 GUI、通过 CLI 打开文件、定位到指定行列等）。

## 环境要求
- 操作系统：Windows 10 1809 或更高版本（推荐 Windows 11）
- Rust 工具链：1.70 或更高版本
- 编译器/SDK：Visual Studio 2022（或已安装 Windows SDK 的构建工具）

说明：
- 仓库通过工作区配置声明了最低 Rust 版本为 1.70，并在工具链文件中固定使用 stable 通道，同时包含 rustfmt 与 clippy 组件，目标平台为 x86_64-pc-windows-msvc。
- 构建默认目标为 MSVC，因此需要 Visual Studio 2022 或等效的 Windows 构建工具链。

## 安装与构建
以下步骤在仓库根目录执行。

- 克隆仓库后，确保系统已安装上述环境要求。
- 构建调试版本（GUI 主程序）：
  - cargo build -p aether-win32 --bin aether-app
- 构建发布版本（GUI 主程序）：
  - cargo build -p aether-win32 --bin aether-app --release
- 构建命令行工具：
  - cargo build -p aether-cli --bin aether

可选：一键构建脚本
- 使用 PowerShell 脚本进行一键构建（支持 --Release 与 --Run 参数）：
  - powershell -File scripts/build-release.ps1
  - powershell -File scripts/build-release.ps1 --Release
  - powershell -File scripts/build-release.ps1 --Release --Run

说明：
- 脚本会先确保 Cargo 可执行文件在 PATH 中，然后调用 cargo build 或 cargo build --release，最后输出产物路径；若传入 --Run，将自动启动 GUI 主程序。

## 运行示例
以下示例均在仓库根目录执行。

- 直接启动 GUI（调试版）：
  - cargo run -p aether-win32 --bin aether-app
- 通过 CLI 启动并打开文件：
  - cargo run -p aether-cli --bin aether -- path/to/file.rs
- 通过 CLI 打开文件并定位到指定行列：
  - cargo run -p aether-cli --bin aether -- file.txt:10:5

说明：
- CLI 会将解析后的启动参数以 JSON 形式传递给 GUI 主进程，由 GUI 负责单实例控制、窗口复用与跳转逻辑。
- 当使用 --wait 时，CLI 会等待 GUI 窗口关闭后再退出。

- 调试版本产物路径：
  - target\x86_64-pc-windows-msvc\debug\aether-app.exe
  - target\x86_64-pc-windows-msvc\debug\aether.exe
- 发布版本产物路径：
  - target\x86_64-pc-windows-msvc\release\aether-app.exe
  - target\x86_64-pc-windows-msvc\release\aether.exe

说明：
- 默认目标三元组为 x86_64-pc-windows-msvc，因此产物位于对应架构子目录下。
- 脚本会在构建完成后打印当前模式下的产物路径，便于快速定位。

## 构建选项说明
- 工作区与包选择
  - -p aether-win32：仅构建 GUI 主程序所在 crate
  - -p aether-cli：仅构建命令行工具所在 crate
  - --bin aether-app / --bin aether：指定要构建的可执行目标名称
- 构建模式
  - 不传 --release：构建调试版本，启用增量编译，便于快速迭代
  - 添加 --release：构建发布版本，启用 LTO、优化等级 3、strip 等，体积更小、性能更好
- 全局构建配置
  - 目标平台：x86_64-pc-windows-msvc（由 .cargo/config.toml 指定）
  - 默认 CPU 特性：sandybridge（由 .cargo/config.toml 指定）
  - 工作区发布配置：启用 fat LTO、codegen-units=1、opt-level=3、panic=abort、strip=true、incremental=false

## 常见问题排查
- 找不到 GUI 主程序
  - 现象：CLI 提示找不到 aether-app.exe
  - 原因：aether.exe 与 aether-app.exe 不在同一目录
  - 解决：确保两者在同一目录，或通过脚本构建后从脚本输出的路径运行
- 无法启动 GUI 主程序
  - 现象：CLI 报错“无法启动 GUI 主程序”
  - 可能原因：目标二进制缺失、权限不足、杀毒软件拦截
  - 解决：确认已正确构建，检查路径与权限
- 单实例行为
  - 现象：再次运行 CLI 未打开新窗口
  - 说明：默认会尝试复用已有窗口并将参数转发给现有实例；如需强制新窗口，可使用 --new-window
- 等待窗口关闭
  - 现象：CLI 立即返回
  - 说明：如需等待 GUI 窗口关闭再退出，请添加 --wait
